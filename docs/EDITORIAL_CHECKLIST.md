# Editorial Review Checklist

Before shipping any hall content or live entry:

## Accuracy
- [ ] Every factual claim has a source (L4 or claims ledger)
- [ ] Dates, vote counts, and dollar figures verified against primary source
- [ ] Case names and citations correct (*Citizens United v. FEC*, 558 U.S. 310)

## Balance (Debate & Reform)
- [ ] Strongest supporter arguments presented fairly
- [ ] Strongest critic arguments presented fairly
- [ ] No straw-man characterizations

## Depth
- [ ] L1 stands alone (no unexplained jargon)
- [ ] L2 adds context without requiring L3
- [ ] L3 includes cross-links to related halls
- [ ] L4 includes full source list

## Live Content
- [ ] Date stamp on live sections
- [ ] Status field current (signatures, litigation, effective dates)
- [ ] Entry JSON and HTML hall in sync

## Accessibility & UX
- [ ] Depth toggle works (all four levels)
- [ ] External links use `rel="noopener"`
- [ ] Headings hierarchical (h1 → h2 → h3)

## Versioning
- [ ] VERSION bumped
- [ ] CHANGELOG updated
- [ ] BUILD_PLAN step status updated
- [ ] Git commit with version tag
